Alcance

Para el caso de Proyectiva, el gateway se limita exclusivamente a la venta digital del producto. Flujo transaccional de pago unico: el usuario ingresa al formulario, realiza el pago y recibe la confirmacion de compra.

Estados del Gateway

LOADEDEn implementacion

Formulario de pago cargado. PayLink generado, usuario aun no realiza el pago.

PENDIENTEEn implementacion

Pago enviado a la pasarela, esperando respuesta.

APROBADOImplementado

Pago exitoso. Venta completada, se procede con emision del producto.

RECHAZADOImplementado

Pago no autorizado por banco, pasarela o antifraude. El usuario puede reintentar.

CANCELADONuevo

Transaccion cancelada activamente por el usuario o el sistema.

EXPIRADANuevo

Sesion vencida por tiempo limite sin completar el pago.

RETURNEDNuevo

El usuario regreso desde el formulario de pago sin completar la transaccion.
