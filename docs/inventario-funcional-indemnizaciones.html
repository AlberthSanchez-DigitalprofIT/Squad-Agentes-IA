Resumen Ejecutivo

Indemnizaciones es la plataforma digital de Seguros Bolívar para la radicación de solicitudes de indemnización por parte de asegurados, beneficiarios, apoderados e intermediarios.

URL DEV: dev-sbindemnizaciones.segurosbolivar.com
Frontend: Angular 16.2.12 (SPA con lazy loading)
Backend: Java 11 + Spring Boot 2.3.4 + MyBatis + Oracle DB
Infraestructura: AWS (S3 + CloudFront + API Gateway + VPC Link + Lambda)
CI/CD: GitHub Actions + SonarQube + JFrog Artifactory
Acceso: Público (sin autenticación para radicar solicitud)
Organización: Tribu Servicios & Operaciones / Squad Operaciones

Tipos de Persona

👤 Persona Natural

Asegurados individuales que solicitan indemnización por coberturas de vida, salud, hogar, movilidad o protección de pagos.

🏢 Persona Jurídica

Empresas aseguradas que solicitan indemnización por coberturas empresariales, patrimoniales y de cumplimiento.

Roles del Solicitante

👤 Asegurado

Titular de la póliza que solicita directamente.

👥 Beneficiario

Persona designada como beneficiaria en la póliza.

💼 Apoderado

Representante legal con poder para actuar en nombre del asegurado.

🤝 Intermediario

Corredor o agente de seguros que gestiona la solicitud.

Inconsistencias UX detectadas

  • 🔴 Salud redirige sin aviso: Al clic en "Salud", navega a otro dominio sin modal ni aviso. Diferente a Hogar/Movilidad que muestran modal.
  • 🟠 Movilidad muestra coberturas inutilizables: Lista 4 coberturas que al clic muestran modal de redirección. Confuso para el usuario.
  • 🟠 Hogar vacía las coberturas: Tras cerrar el modal, la sección queda vacía sin forma de regresar.
  • 🟠 Sin indicador visual de canal: Los iconos del menubar no distinguen entre ramos con formulario y ramos que redirigen.
  • 🟢 Persona Jurídica funcional: Las 10 coberturas de Empresas se muestran correctamente.

Hallazgos de la Exploración

  • 🔴 CORS bloqueado en fuente corporativa: La fuente bolivar.ttf desde CloudFront está bloqueada por CORS. La tipografía corporativa no se muestra en DEV.
  • 🔴 Fuentes Roboto corruptas: Todas las variantes (WOFF2, WOFF, TTF) fallan con "invalid sfntVersion". Archivos corruptos o mal servidos.
  • 🔴 Configuración de documentos por cobertura: El paso 3 para "Gastos Médicos" pide documentos genéricos (Matrimonio, Nacimiento, Divorcio) que no aplican a esa cobertura. Posible error de configuración backend.
  • 🟠 Repos "clever-*" inexistentes: Los 4 repos proporcionados no existen en GitHub. Los repos reales usan prefijo comunes-.
  • 🟠 GTM bloqueado por ORB: Google Tag Manager bloqueado por Cross-Origin Read Blocking en DEV.
  • 🟠 Imágenes footer SFC no cargan: vigilado_desktop.png y vigilado_mobile.png generan múltiples errores ERR_ABORTED.
  • 🟠 Checkboxes PrimeNG interceptan clicks: Los .p-checkbox-box interceptan pointer events, dificultando automatización E2E y accesibilidad.
  • 🟠 Date picker no acepta input manual: Solo funciona vía calendario visual, no como input editable.
  • 🟢 Formulario pasos 1-2 funcionales: Validaciones estrictas correctas. Botón "Continuar" condicional.
  • 🟢 Carousel de coberturas operativo: 6 categorías con 14+ coberturas.
  • 🟢 APIs versionadas: /api/v1/ cumple estándar org. Patrón reference-data limpio.
  • 🟢 Sin captchas ni bloqueos: DEV sin restricciones adicionales para testing.
